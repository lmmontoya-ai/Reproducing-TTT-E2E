# Prime 125M H100 Smoke

Date: 2026-03-12

Environment:
- Prime on-demand pod `7a257eb346ef40c2acb2c3450414c611`
- `8x H100 80GB`
- Price at launch: `$14.40/hr`

Outcome:
- The rebased parity `jax_train` path did not fit the paper config out of the box.
- Successful pretrain smoke required three runtime changes:
  - local multi-GPU data parallelism with `eqx.filter_pmap`
  - memory-efficient attention via `jax.nn.dot_product_attention`
  - honoring `training.accum_steps`
- The first working recipe was `S0_PRETRAIN_FA_125M` with `training.accum_steps=8`.

Successful smoke runs:
- `smoke_125m_gpu7`
  - stage: `S0_PRETRAIN_FA_125M`
  - steps: `1`
  - tokens seen: `524288`
  - runtime event elapsed: `87.58s`
  - wall time: `107.33s`
  - W&B: `https://wandb.ai/luism31/ttt-e2e-warmstart/runs/2g0pa8lf`
- `smoke_125m_gpu8`
  - stage: `S0_PRETRAIN_FA_125M`
  - steps: `2`
  - tokens seen: `1048576`
  - runtime event elapsed: `81.13s`
  - wall time: `100.41s`
  - W&B: `https://wandb.ai/luism31/ttt-e2e-warmstart/runs/lgwml1qc`

Interpretation:
- The current parity runtime is launchable only with the `accum_steps=8` workaround.
- Measured throughput is too low for the full exploratory 125M ladder under the current `$100` budget cap.
- Using the observed `81.13s / 2 steps`, the pretrain stage alone would be on the order of `~54 hours` for `4800` steps, before adaptation, extension, or eval.
